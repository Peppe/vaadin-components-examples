v-grid Columns

Defining Columns

Columns can be defined using HTML or Javascript.

For the HTML version, the table element might seem redundant, but it is required in order for the browsers to parse the contained markup correctly. We also need to provide some data for the grid, using tbody, in order to render something.

Read the column API documentation to see all the attributes and properties you can define for a column.

One Two
grid.columns = [ { headerContent: "First column" }, { headerContent: "Second column" } ];

Adding Columns Dynamically

Columns can be added dynamically using the JavaScript API. You can also modify the lightDom of the component, but it is not the recommended way.

grid.data.source = [["One", "Two"]]; grid.addColumn({ headerContent: "Column one" }); // Specify the index or name of the column before which to add the new column grid.addColumn({ headerContent: "I’m First"}, 0);

Column width

By default all the columns share the available size equally. They may also be assigend an explicit width, a max width and a min width.

 grid.data.source = myBigData; // Setting the width properties with JS api grid.columns[0].width = 30; grid.columns[1].maxWidth = 200; grid.columns[2].minWidth = 100;
