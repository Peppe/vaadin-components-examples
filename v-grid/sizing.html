v-grid Sizing and Styling

Sizing the grid

If Grid is not assigned any bounds externally it will size automatically by itself. By default it will take 100% of the horizontal space available and its default height depends on the number of contained data rows.

Standard CSS rules can be used to set a size for the grid or alternatively Grid can be assigned a pre-defined height (in rows) either declaratively in the DOM API or programmatically with the JS API

 grid.data.source = myBigData; // Setting the explicit row count in JavaScript grid.rows = 5;

Styling Rows

Grid data rows can be labeled with individual class names by setting a rowClassGenertor.

 grid.data.source = myBigData; grid.rowClassGenerator = function(row) { var activity = row.data[2]; return 'activity-' + activity.toLowerCase(); };

Styling Cells

Single data cells can be styled with the help of a cellClassGenertor.

 grid.data.source = myBigData; grid.cellClassGenerator = function(cell) { if (cell.index == 2) { var activity = cell.row.data[2]; return 'activity-' + activity.toLowerCase(); } return ''; };

Single data cells can be styled with the help of a cellClassGenertor.

'white' : 'black'; };
